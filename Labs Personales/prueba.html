¿Cuál es la diferencia entre Internet y la World Wide Web?

La World Wide Web es una red de páginas web que se encuentran por sus direcciones. Internet es la red de redes que constituye la base de la web. Internet es una red mundial de miles de millones de ordenadores y otros dispositivos electrónicos. (UAB, 2019)

¿Cuáles son las partes de un URL?

La URL (Localizador Uniforme de Recursos). Una URL es una dirección web utilizada para identificar recursos en la web. Estas son las partes que conforman una URL Protocolo - Que incluye HTTP, HTTPS, Protocolo de Transferencia de Archivos (FTP) , Protocolo Simple de Transferencia de Correo (SMTP). Dominio - Nombre que se utiliza para identificar una o varias direcciones IP donde se encuentra el recurso. Ruta - Especifica la ubicación del recurso en el servidor. Parámetros - Datos adicionales utilizados para identificar o filtrar el recurso en el servidor. (Aviani, 2019)

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

El método Get se utiliza para recuperar u obtener la información del servidor dado utilizando una URL determinada. El método Post se utiliza para insertar nuevos elementos en el servidor backend. El método PUT se utiliza más a menudo para actualizar un recurso existente. El método PATCH se utiliza para actualizar los valores de las propiedades del recurso. El método DELETE se utiliza para eliminar un recurso especificado por su URI. El método HEAD es muy similar al get, pero sólo transfiere la línea de estado y la sección de cabecera. (Rafiq, 2020)

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Cuando se ingresa al formulario se utiliza el método GET debido a que se tiene que obtener la información del servidor. Una vez contestado el formulario se utiliza el método PUT para actualizar la información.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

Cuando se accede a una página a través de un URL se utiliza el método GET.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Un código de estado HTTP 200 significa éxito. El cliente ha solicitado documentos al servidor. El servidor ha respondido al cliente y le ha entregado los documentos. (Griffth, 2021).

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Cuando ocurre este error no es responsabilidad del desarrollador de arreglarlo. Un código de estado HTTP 404 significa básicamente que la URL solicitada simplemente no se encontró. Al reportar un error 404 podría significar que la página simplemente no existe, o también podría causarla cuando el usuario escribe mal la URL. Esto podría "arreglarse" recomendando enlaces populares que sean similares al que el usuario solicitó. (Griffth, 2021).

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

El código de estado 500 "error interno del servidor" es un error en el lado del servidor. Esto significa que el servidor encontró algo inesperado que le impidió cumplir con la solicitud. Esto podría ser ambiguo debido a la naturaleza de únicamente notificar que algo está mal, pero no señalar la raíz exacta del problema en el servidor. En este caso, el desarrollador es responsable de solucionar el problema lo antes posible. (Griffth, 2021)

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Los elementos obsoletos son aquellos que se permiten, pero no se recomiendan, y están siendo sustituidos por otros más nuevos. Algunos atributos de HTML4 que ya no se permiten en HTML5 y que se han eliminado por completo son los siguientes: img e iframe. caption, iframe, img, input, object, legend, table, hr, div, h1, h2, h3, h4, h5, h6, p, col, colgroup, tbody, td, tfoot, th, thead y tr. (whwhen, 2022)

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

Nuevos elementos estructurales

En el directorio de HTML5 se añaden bastantes elementos estructurales, a diferencia de HTML4. Algunos de ellos como article, aside, bdi, details y section. (Kumari, 2015)

Declaración DOCTYPE

Con HTML 4 era necesario indicar al navegador la definición del tipo de documento y con HTML5 basta con un simple !DOCTYPE html. (Kumari, 2015)

Nuevo atributo input "entrada" del formulario 'Required'

Este nuevo atributo fue añadido como tipo de entrada en HTML5 en comparación con HTML4. (Kumari, 2015)

¿Qué componentes de estructura y estilo tiene una tabla?

Componentes de una mesa Filas: las filas van horizontalmente de izquierda a derecha. Columnas - las columnas van verticalmente de arriba a abajo Celdas - las celdas componen el espacio donde se pueden almacenar los datos. La etiqueta table es una etiqueta contenedora - su propósito es mantener los otros componentes de la etiqueta. La etiqueta table tiene una etiqueta de cierre obligatoria /table. Etiqueta tr - para las filas Las columnas no tienen una etiqueta específica Etiqueta td o th. Una etiqueta td representa los "datos de la tabla" (inserta los datos en la tabla). Una etiqueta th representa el "encabezado de la tabla". (Fast Web Start, 2020)

¿Cuál es el ciclo de vida de los sistemas de información?

El ciclo de vida de los sistemas de información se compone de siete etapas diferentes. Este ciclo sigue las siguientes etapas: planificación, análisis de requisitos, diseño, desarrollo, prueba, integración y ejecución y, por último, operación y mantenimiento. (Warbleton Council, 2020)
